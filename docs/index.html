Mapeamento da Acessibilidade aos Alimentos Saudáveis em Favelas e Comunidades Urbanas Brasileiras

Author

Marcio Antonio Pizzonia Junior

Published

May 14, 2026

Visão Geral

O presente estudo visa prover mapas gerados pela Função de Densidade de Kernel sobre o acesso aos alimentos saudáveis em favelas e comunidades urbanas, por meio da obtenção, processamento, geocodificação e classificação dos dados dos CNPJs provenientes da Receita Federal Brasileira (RFB), como também os dados sobre os estabelecimentos classificados como de venda de alimentos saudáveis do OpenStreetMap (OSM), via Overpass API.

A classificação sobre a saudabilidade alimentar segue o estudo Locais-Nova, usando o Grupo 0 (G0), este não sendo parte da classificação original, mas sim um subgrupo existente no grupo G1 + G2, sendo é composto por estabelecimentos que vendem alimentos exclusivamente saudáveis que deveriam fazer parte da dieta populacional.

Para instruções de como usar o pipeline, veja README no repositório.

Problema

O projeto AcessoSAN almeja desenvolver novos métodos de medição e de análise da desilguadade existente no acesso aos alimentos saudáveis, particularmente nas favelas e comunidades urbanas brasileiras. Para alcançar tais objetivos, faz-se necessário o uso de bases de dados confiáveis e atualizadas sobre os estabelecimentos que vendem comida saudável.

A RFB lista companhias que operam no Brasil, incluindo varejos alimentares. Entretano, não fornece informações geocodificadas, necessárias para a análise espacial. Esse pipeline segue os trabalhos realizados pelos estudos de Daniel Vartanian et al. (2026) para a georreferenciar os dados dos CNPJs da RFB e de Flávio Soares de Freitas et al. (2026), para obter as métricas de acessibilidade do grupo G0.

Além disso, para entender melhor os gaps informacionais existentes sobre o acesso aos alimentos saudáveis em favelas e comuinidades urbanas, ralizar-se-á, também, análises referentes aos dados disponíveis na Overpass API como meio de comparação com os resultados obtidos com a RFB.

Métodos

Fontes dos Dados

Os dados usados nesse repositório advém das seguintes fontes:

  • Receita Federal Brasileira (RFB):
    • Dados de pessoas jurídicas do Cadastro Nacional da Pessoa Jurídica (CNPJ) base de dados usados para identificar empresas (Brasil et al., n.d.).
    • Códigos de agências e municípios brasileiros (Tabela de Órgãos e Municípios, TOM) (Brasil & Sistema de Vigilância Alimentar e Nutricional, n.d.), usados para identificar os municípios na base de dados dos CNPJs.
  • Instituto Brasileiro de Geografia e Estatística (IBGE):
    • Códigos e descrições da Classificação Nacional de Atividade Econômicas (CNAE) (Instituto Brasileiro de Geografia e Estatística & Comissão Nacional de Classificação, n.d.), usado para correspondência com os grupos Locais-Nova.
    • Dados dos municípios brasileiros via geobr, pacote R (Pereira & Goncalves, n.d.) usado para validação cruzada.
    • Base de dados espaciais aberta dos muninípios brasileiros disponível no Cadastro Nacional de Endereços para Fins Estatísticos (CNEFE) via geocodebr, pacote R (Pereira & Herszenhut, n.d.) usado para geocodificação dos endereços.
    • Dados dos limites dos setores censitários provenientes do Censo 2022 e daddos dos limites municipais por meio do geobr, pacote R (Pereira & Goncalves, n.d.) usado para análise espacial e cálculo da densidade populacional.
    • Dados populacionais do Censo 2022 via censobr, pacote R (Pereira & Goncalves, n.d.), usado para calcular a densidade populacional.
  • Centro de Estudos da Metrópole (CEM):
    • Tabela de Correspondência contendo a classificação que relaciona os códigos CNAEs dos estabelecimentos aos grupos Locais-Nova (Fernandes & Giannotti, 2026).
    • Tabela de Correspondência fornecendo padronização de endereços para fins de geocodificação (Vartanian et al., 2026).
  • Mapzen Terrain Tile
    • Dados altiimétrocos (Mapzen, n.d.) via elevatr, pacote R (Hollister et al., 2025) usado para cálculo do tempo de viagem e métricas de acessibilidade.
  • OpenStreetMap (OSM):
    • Dados espaciais de vias e caminhos para pedestres (OpenStreetMap Contributors, n.d.), obtidos por meio do projeto Geofabrik (Geofabrik GmbH, n.d.), utilizados para o cálculo dos tempos de deslocamento.
  • PMTilesBR: Dados Geoespaciais em Mosaico para o Brasil
    • Dados dos municípios brasileiros no formato PMTiles (Vartanian et al., 2026), utilizados para a representação de informações geográficas em mapas interativos com MapLibre (MapLibre Contributors, 2022), via mapgl, pacote R (Walker, 2026).
  • Tecnologia da Uber:
    • Dados de malhas hexagonais estruturados a partir do sistema hierárquico de indexação geoespacial hexagonal (H3) (Uber Technologies, n.d.), processados com os pacotes h3r (Cooley & Shao, 2024) e h3jsr (O’Brien, 2023), pacotes R empregados na criação de grades hexagonais para análise e visualização espacial.

Processamento dos Dados

O processamento dos dados segue o workflow de ciência de dados criado por Wickham et al. (2023) como ilustrado in Figure 1. Todos os processos foram feitos usando o sistema de publicação Quarto em conjunto com as linguagens de programação AWK (Aho et al., 2023) e R (R Core Team, n.d.), apoiadas por diverso pacotes do R.

Para o workflow e o processamento dos dados foi dado priodridade aos pacotes do tidyverse, rOpenSci e r-spatial frameworks, assim como outros pacotes que aderem ao manifesto tidy tools (Wickham, 2023).

Matriz de Deslocamento

A matriz de deslocamento é computada por meio do pacote r5r do R (Pereira et al., 2021), que realiza interface com o mecanismo de roteamento R5. Esta ferramenta possibilita a estimação detalhada de tempos de viagem a partir de redes de transporte multimodais, incorporando tipos de vias, condições de tráfego e variações altimétricas. Os tempos de deslocamento são obtidos do OpenStreetMap (OSM) a partir de dados de vias e caminhos de pedestres (OpenStreetMap Contributors, n.d.) e de dados altimétricos do conjunto Mapzen Terrain Tile (Mapzen, n.d.).

Estilo do Código

O Tidyverse manifesto tidy tools (Wickham, 2023), guia do estilo de código (Wickham, n.d.-a) e design principles (Wickham, n.d.-b) foram seguidos para garantir consistência e melhorar a legibilidade.

Reprodutibilidade

O pipeline é integralmente reprodutível e pode ser reexecutado a qualquer momento. Para assegurar consistência nos resultados, utilizou-se o pacote renv package (Ushey & Wickham, 2025) para gerenciar e restaurar o ambiente R. Consulte o arquivo README no repositório para instruções de execução.

Check Parameters

The pipeline main parameters are defined in the YAML header of its Quarto document.

The year and month variables represent the year and month of the consolidated RFB CNPJ dataset.

The municipality_codes variable identifies the IBGE codes of the municipalities included in the analysis. Click here to see a list of all Brazilian municipalities and their codes.
